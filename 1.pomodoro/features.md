# ポモドーロタイマー実装機能一覧

このドキュメントは、ポモドーロタイマーアプリケーションを実装するうえで必要な機能を整理したものである。

## 1. MVPとして必須の機能

### 1.1 タイマー基本操作

- 作業セッションを開始できる
- 実行中のタイマーを一時停止できる
- 一時停止したタイマーを再開できる
- 現在のフェーズを初期状態にリセットできる

### 1.2 タイマー表示

- 現在のフェーズを表示できる
  - 作業中
  - 短休憩
  - 長休憩
- 残り時間を mm:ss 形式で表示できる
- 経過率に応じて円形プログレスを更新できる

### 1.3 フェーズ遷移

- 作業セッション終了後に休憩フェーズへ自動遷移できる
- 規定回数までは短休憩へ遷移できる
- 規定回数到達時は長休憩へ遷移できる
- 休憩終了後に次の作業フェーズへ戻れる

### 1.4 設定管理

- 現在の設定値を取得できる
- 設定値を更新できる
- 以下の設定を保持できる
  - 作業時間
  - 短休憩時間
  - 長休憩時間
  - 長休憩に入るまでの作業回数

### 1.5 実績記録と表示

- 作業セッション完了時に実績を記録できる
- 今日の完了回数を表示できる
- 今日の集中時間合計を表示できる

## 2. 内部的に必要な機能

### 2.1 タイマー状態管理

- 現在フェーズを保持できる
- 実行中か停止中かを保持できる
- 残り時間を保持できる
- 完了済み作業回数を保持できる
- 開始時刻と終了予定時刻を保持できる

### 2.2 時刻ベースの残り時間計算

- 毎秒減算ではなく終了予定時刻との差分で残り時間を計算する
- バックグラウンド復帰時のズレを補正できる
- 画面再読み込み後に状態復元しやすい形で計算できる

### 2.3 状態復元

- ブラウザ再読み込み後にタイマー状態を復元できる
- 少なくとも現在フェーズ、終了予定時刻、完了済み回数を復元できる

### 2.4 バリデーション

- 設定値が整数であることを検証できる
- 設定値が 0 より大きいことを検証できる
- 不正な入力に対してエラー応答を返せる

## 3. APIとして必要な機能

### 3.1 画面表示

- GET / でタイマー画面を返せる

### 3.2 統計取得

- GET /api/stats/today で今日の実績を返せる

### 3.3 設定取得・更新

- GET /api/settings で現在の設定を返せる
- PUT /api/settings で設定を更新できる

### 3.4 セッション記録

- POST /api/sessions で完了した作業セッションを保存できる

## 4. データ保存として必要な機能

### 4.1 設定データ

- 作業時間を保存できる
- 短休憩時間を保存できる
- 長休憩時間を保存できる
- 長休憩間隔を保存できる

### 4.2 セッション履歴データ

- セッション種別を保存できる
- 集中時間を秒単位で保存できる
- 完了日時を保存できる

### 4.3 日次集計

- 初期実装では sessions から当日分を集計できる
- 必要になれば日次集計テーブルへ拡張できる

## 5. 実装時に分離しておくべき責務

### 5.1 サーバー側

- 画面配信
- API提供
- 設定保存
- 実績保存
- 当日統計集計

### 5.2 クライアント側

- タイマー進行管理
- 残り時間計算
- DOM更新
- 円形プログレス描画
- API通信
- ローカル状態保存

## 6. 品質確保のために必要な機能

### 6.1 テストしやすい構造

- タイマー状態遷移を純粋関数として実装する
- 進捗率計算を純粋関数として実装する
- 設定値バリデーションを純粋関数として実装する
- 現在時刻を注入可能にする

### 6.2 テスト対象

- タイマー状態遷移
- 長休憩判定
- 残り時間計算
- 進捗率計算
- 設定値バリデーション
- 設定APIの正常系と異常系
- セッション記録APIの正常系と異常系
- SQLiteへの保存処理
- 今日の集計処理

## 7. 実装優先順位

1. タイマー表示と基本操作
2. フェーズ遷移と長休憩判定
3. 設定取得と設定保存
4. セッション記録と今日の進捗表示
5. 状態復元
6. テスト整備
